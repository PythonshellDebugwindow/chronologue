Introduction

Sound change applicators work by applying a list of rules to a word. They can be used to estimate pronunciation, derive grammatical forms, or simulate historical sound changes.

Rule structure

Rules are made up of up to five segments, separated by slashes. The general structure of a rule is as follows:

target/change/environment/exception/else

The target segment specifies what to search for within a word.

The change segment specifies what to replace the target with.

The environment segment specifies what condition(s) must be fulfilled in order for a change to occur.

The exception segment specifies what condition(s) must not be fulfilled in order for a change to occur.

The else segment specifies an alternative change to be used if the exception is fulfilled.

To summarise, a rule replaces the target with the change if the environment is fulfilled, unless the exception is fulfilled, in which case it will be replaced by the else instead.

All segments can be empty. An empty target means that the change or else should simply be inserted; an empty change or else means that the target will be deleted; and an empty environment or exception will be ignored.

Each rule should appear on a separate line. Lines without a slash are ignored, and can be used for annotating long rulesets.

Categories

Categories are sets of sounds which can be referenced together in a rule. They can be edited on the Edit Categories page.

All categories have a name (a single uppercase letter) and a list of members (letters or phones). Categories can either be phonological or orthographical; the former are used in pronunciation estimation, while the latter are used by grammar tables.

While not strictly necessary, categories can greatly reduce the length of rulesets. For example, if the category N contains the members m,n,ŋ, then the rule N/m/_p would change all three nasals to m before p.

Examples

The following examples should give a general idea of ChronoSCA's syntax.

Rule Meaning Example
d/t Change all d to t detado → tetato
#/#ai Suffix -ai aider → aiderai
C/h/V_V Change all C to h between two V tapa → taha
C//_# Delete C word-finally apat → apa
/ə/C_C Insert ə between two C aptak → apətak
/ə/C_# Insert ə after word-final C apat → apatə

Local replacement

Local replacement rules are used to replace all instances of text.

Simple replacement

To replace constant text with constant text, use a slash.

For example, p/f will replace all instances of p with f.

Category replacement

To replace all members of a category with constant text, use a slash. For example, V/ə will replace all members of V with ə.

It is also possible to perform a 1-to-1 replacement of category members. For example, if P is p,t,k and B is b,d,g then P/B will replace p with b, t with d, and k with g.

In a category replacement, if the change or target contains a category name, it must not contain anything else. For example, P/ʰP and Bh/P would be invalid rules.

Global rules

Global rules are used to add affixes, and to perform global replacements.

Affixes

Affix rules use # as a target, and a #-separated prefix and suffix as a change.

For example, #/#ing will add the suffix ing, and #/ge#t will add the prefix ge and the suffix t.

Global replacements

Global replacements can be used to replace the entire word. For example, #/abc will replace the entire word with abc.

They can also be used to perform word-initial or word-final text replacements. For example, er#/ons will replace word-final er with ons, and #h/ will delete h word-initially.

Categories are not supported in global replacements, except in conditions.

Conditions

There are two types of conditions: local and global. Affix and global replacement rules require global conditions, while local replacement rules can use either type.

Local conditions

Local conditions check the target's local environment (e.g., whether it is before or after a certain letter).

In local conditions, the character _ indicates the target, and # marks a word boundary. For example:

  • s_ means that the target must be directly after s.
  • _V means that it must be directly before a member of V.
  • #p_r means that the target must be between p and r, and that the p must be at the start of the word.

Global conditions

Global conditions check the word as a whole (e.g., whether it starts with a certain letter). Any condition without _ is global.

# indicates a word boundary. For example:

  • x means that the word must contain x.
  • #st means that the word must start with st.
  • V# means that the word must end with a member of V.

Environment, exception, and else

The environment and exception sections of a rule are pipe-separated lists of conditions. For one of these sections to be fulfilled, at least one of its conditions must be satisfied.

For example, o/u/w_|_# would change o to u after w or word-finally.

The else must be of the same form as the change; for example, the rule B/P/V_V//h would be invalid.

Miscellaneous

Special characters

Uppercase letters and /#_|%,! are considered special, meaning that they are treated differently from regular text. They can be escaped (treated as normal text) by prepending an exclamation mark; for example, the rule #/#!A would append an uppercase A.

Referencing the target

In segments other than the target, the percent sign % refers to the text being replaced.

For example, the rule V/%%/_# would duplicate word-final vowels, #/%% would duplicate the word, and V//%_ would delete vowels which occur after a copy of themselves (so it would change aabaaa to aba, and leave ae unchanged).

Percent signs cannot appear in conditions in global rules.

Temporary categories

Temporary categories are categories which only exist in a single segment. They remove the need to assign letters to one-off categories.

A temporary category is indicated using commas. For example, the rule m,n,ŋ/b,d,g/_# would change m to b, n to d, and ŋ to g word-finally.

Temporary categories can only be used as target, change, or else segments. All members of a temporary category must be normal text (no special characters are allowed, unless escaped), and they must all be non-empty.

Multi-character category members

Category names in rules always refer to the longest possible category member which appears starting at the relevant position in the word, even if this causes a condition to not be satisfied.

This means that potentially unexpected results can occur if multiple members of a category start or end with the same letters (such as s,sh or s,hs), and that category is used in a rule with an environment or exception segment.

For example, if C contains s and sh, then the rule a/e/_Ch would not change the word ash to esh; this is because the C in the rule would match sh rather than the shorter s, despite the condition not being satisfied as a result. (For the same reason, the rule would change the word ashh to eshh.)

This also applies if the category is the target of the rule; for example, if C contains s,sh, then C//e_h would not change esh to eh, but would change eshh to eh.
